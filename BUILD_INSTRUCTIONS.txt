GENIE_LAMP BUILD & INSTALLATION

OVERVIEW:
Standalone gesture detector with config file support and systemd service.
- NO rmkit dependencies
- Reads /dev/input/event2 directly
- Config file at /opt/etc/genie_lamp.conf
- Runs as systemd service on boot

QUICK START:
1. cd src/genie_lamp
2. make              # Build
3. make install      # Deploy binary + config + service
4. make enable       # Enable service (start on boot)
5. make start        # Start service now

MAKEFILE TARGETS:
  make             Build genie_lamp
  make install     Deploy binary, config, and service file
  make enable      Enable service (auto-start on boot)
  make start       Start service now
  make stop        Stop service
  make status      Check service status
  make logs        View service logs (live)
  make clean       Remove binary

CONFIGURATION:
Edit /opt/etc/genie_lamp.conf to add gestures:

  # Example: 3-finger tap draws square
  gesture=tap
  fingers=3
  command=echo -e "pen down 500 500\npen move 900 500..." | /opt/bin/lamp

  # Add more gestures separated by blank lines

MANUAL RUN (for testing):
  ssh root@10.11.99.1 /opt/bin/genie_lamp /opt/etc/genie_lamp.conf

SERVICE MANAGEMENT:
  systemctl status genie_lamp    # Check status
  systemctl start genie_lamp     # Start
  systemctl stop genie_lamp      # Stop
  systemctl restart genie_lamp   # Restart after config change
  journalctl -u genie_lamp -f    # View logs

TECHNICAL DETAILS:
- Pure C++ with only Linux input API
- Config file parser supports: gesture=tap, fingers=N, command=...
- Simple multitouch gesture detection
- No framebuffer, no rmkit UI framework
- 254 lines of standalone code

FILES:
  main.cpp              Standalone implementation
  genie_lamp.conf       Example config (3-finger tap = square)
  genie_lamp.service    Systemd service file
  Makefile              Build & deployment

DEPENDENCIES:
  arm-linux-gnueabihf-g++ (ARM cross-compiler)
  Linux kernel headers
  That's it!
