GENIE_LAMP BUILD INSTRUCTIONS (STANDALONE VERSION)

OVERVIEW:
genie_lamp is now a standalone C++ program with NO rmkit dependencies.
It reads touch events directly from /dev/input/event2 and detects gestures.

BUILD:
1. cd src/genie_lamp
2. make
3. Binary: genie_lamp (in same directory)

DEPLOY:
  scp src/genie_lamp/genie_lamp root@10.11.99.1:/opt/bin/

RUN:
  ssh root@10.11.99.1 /opt/bin/genie_lamp

WHAT IT DOES:
- Detects 3-finger tap gesture
- Draws a square using lamp when detected
- No configuration file needed
- No framebuffer dependencies
- No rmkit UI framework dependencies

TECHNICAL DETAILS:
- Pure C++ with only Linux input API (<linux/input.h>)
- Opens /dev/input/event2 directly
- Processes multitouch events (ABS_MT_* events)
- Simple gesture detection without UI framework overhead
- Hardcoded lamp command: draws 200x200 square at (500,500)

DEPENDENCIES:
- arm-linux-gnueabihf-g++ (ARM cross-compiler)
- Linux kernel headers (for input.h)
- That's it! No rmkit, no okp, no complex build system.

FILES:
- src/genie_lamp/main.cpp: Complete standalone implementation (160 lines)
- src/genie_lamp/Makefile: Simple direct compilation

OLD FILES (for reference):
- main.cpy.old: Old rmkit-based version
- gesture_parser.cpy.old: Old rmkit gesture parser
- Makefile.rmkit: Old rmkit Makefile

WHY STANDALONE:
The original approach used rmkit's UI framework which caused:
- Need to build rmkit.h first
- Undefined references to UI symbols
- Complex okp transpilation
- Difficult debugging

This standalone version eliminates ALL of that complexity.
