GENIE_LAMP BUILD INSTRUCTIONS

1. Go to rmkit directory:
   cd resources/rmkit

2. Build:
   make genie_lamp TARGET=rm

3. Binary will be at:
   src/build/genie_lamp

4. Deploy:
   scp src/build/genie_lamp root@10.11.99.1:/opt/bin/
   scp ../../src/genie_lamp/genie_lamp.conf root@10.11.99.1:/opt/etc/

5. Run:
   ssh root@10.11.99.1 /opt/bin/genie_lamp /opt/etc/genie_lamp.conf

WHAT WAS CHANGED FROM ORIGINAL GENIE:
- Removed framebuffer::get() calls (gesture_parser.cpy build_swipe_gesture and build_tap_gesture)
- Removed ui::MainLoop::refresh() and redraw() from main loop (main.cpy)
- Added fixed dimensions: SCREEN_WIDTH 1404, SCREEN_HEIGHT 1872
- No rm2fb or fbink dependencies
- Added gesture_parser.cpy to FILES variable in Makefile (fixes undefined references)

FILES:
- src/genie_lamp/main.cpy: Minimal event loop
- src/genie_lamp/gesture_parser.cpy: Fixed dimensions, no FB
- src/genie_lamp/genie_lamp.conf: Test config (3-finger tap = square)
- src/genie_lamp/Makefile: FILES=main.cpy gesture_parser.cpy

BUILD FIX:
The Makefile must include BOTH main.cpy and gesture_parser.cpy in FILES variable.
Original genie works with only main.cpy due to okp auto-detection, but genie_lamp
requires explicit listing to avoid undefined reference errors during linking.

DO NOT build from src/genie_lamp - must build from resources/rmkit
